Visualizer Formulas & Math

Computer Networks · Reference

How the delay values are calculated.

This page shows the exact equations used by the visualizer and how each input maps to the symbols in the formulas.

Main Equation

Total End-to-End Delay

dtotal = (h + 1) × (dtrans + dprop) + h × (dproc + dqueue)

Hop count is the number of intermediate devices the traffic passes through before reaching the destination. That means there are h + 1 links between source and destination, but only h intermediate devices that contribute processing and queueing delay.

Symbols

What Each Variable Means

h Number of intermediate devices on the path
L Packet length in bits
R Link rate in bits per second
d Distance per link
s Propagation speed
c Speed of light, about 3.00 × 108 m/s

Component 1

Transmission Delay

dtrans = L / R

Transmission delay is the time required to push all packet bits onto the link. In the app:

L = packet size in bytes × 8

R = bandwidth in Mbps × 1,000,000

Component 2

Propagation Delay

dprop = d / s

Propagation delay is the time required for the signal to travel across the physical medium. In the app:

d = distance per link in km × 1,000

s = propagation factor × c

Component 3

Processing Delay

dproc = processing input in ms / 1000

Processing delay represents router work such as checking headers, making forwarding decisions, and preparing the packet for the next link. The input is entered in milliseconds and converted to seconds internally.

Component 4

Queueing Delay

dqueue = queueing input in ms / 1000

Queueing delay represents time spent waiting in buffers before the packet can be transmitted. This is also entered directly in milliseconds and converted to seconds internally.

Worked Example

Using the Default Values

Inputs

Packet size = 1500 bytes

Bandwidth = 100 Mbps

Distance per link = 250 km

Propagation factor = 0.67c

Hop count = 4 devices

Processing = 0.3 ms per device

Queueing = 1.2 ms per device

Transmission

L = 1500 × 8 = 12000 bits

R = 100 × 106 bps

dtrans = 12000 / 100000000 = 0.00012 s = 0.120 ms

Propagation

d = 250000 m

s = 0.67 × 299792458 ≈ 200860947 m/s

dprop ≈ 250000 / 200860947 ≈ 0.001245 s = 1.245 ms

Per-Link Travel

0.120 ms + 1.245 ms

= 1.365 ms per link

Per-Device Delay

0.300 ms + 1.200 ms

= 1.500 ms per device

End-to-End Total

(5 × (0.120 + 1.245)) + (4 × (0.300 + 1.200)) ms

= 12.825 ms total

Path Model

Why the Total Uses Multiplication

Basic calculations assume each link is identical and each intermediate device is identical. With per-link values, the total is calculated as a sum across links and devices:

dtotal = Σ(dtrans,link i + dprop,link i) + Σ(dproc,device j + dqueue,device j)
